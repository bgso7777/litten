### 발생한 문제와 해결책

| 문제 | 원인 | 해결 |
|------|------|------|
| YouTube Error 153 | 기본 WebView UA가 YouTube에 차단됨 | Chrome Mobile UA + youtube.com/embed 대신 m.youtube.com 로드 |
| "...더보기" 버튼 없음 | youtube_player_flutter 위젯은 YouTube 네이티브 UI 미포함 | m.youtube.com 전체 페이지 WebView로 교체 |
| Description 패널 스크롤 불가 | DraggableScrollableSheet가 수직 드래그 제스처 가로챔 | `WebViewWidget(gestureRecognizers: {Factory<VerticalDragGestureRecognizer>(...)})`  |
| "스크립트 표시" 메뉴 없음 | JS 숨김 목록에 `ytm-menu-renderer` 포함 | `ytm-menu-renderer` 숨김 목록에서 제거 |
| `_hideScript` 빌드 오류 | static const 내 `initState` 클로징 브레이스 오삽입 | 배열 join 방식으로 CSS 문자열 구성, 브레이스 위치 수정 |
| `Factory` 미정의 오류 | `flutter/foundation.dart` import 누락 | `import 'package:flutter/foundation.dart'` 추가 |

### 얻은 교훈
1. YouTube WebView 임베드 URL(`/embed/`)은 Android WebView에서 Error 153으로 차단됨. 반드시 `m.youtube.com/watch?v=` + Chrome UA 사용
2. `DraggableScrollableSheet` 내부 WebView는 기본적으로 스크롤 제스처를 받지 못함. `gestureRecognizers`로 명시적 위임 필요
3. YouTube 모바일 페이지 JS 숨김 처리 시, 숨기고 싶지 않은 기능(⋯ 메뉴 등)이 같은 컴포넌트에 포함될 수 있으므로 신중하게 선택해야 함
