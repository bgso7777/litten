### 작업 시 주의 사항

- **백엔드 컴파일**: `./build.sh clean package -DskipTests` 사용 (기본 mvn은 JDK 25로 Lombok 깨짐).
  성공 판정은 로그의 `Compiling N source files` 확인으로.
- **배포 순서**: DB 마이그레이션 먼저 → 톰캣 **정지**(`pgrep -f "catalina.startup.Boot[s]trap"` 로 0 확인) → WAR 교체 → 기동.
  가동 중 WAR를 올리면 autoDeploy 개입으로 앱 소실 사고 전례 있음.
- **실제 DB는 원격**(`www.litten7.com`). 로컬 `litten7` DB는 껍데기.
- **아이폰 설치**: `xcrun devicectl device install app` (flutter install은 데이터 초기화).
  설치 전 `transportType`으로 유선 기기인지 확인. 두 기기(소병규의 iPhone=11, 지니=14) 모두 프로파일 등록됨.
- **패키지명**: 안드로이드 `com.litten.app` / iOS `com.sobyungkyu.litten`
- **디스크 자주 100% 도달** — 릴리즈 빌드마다 3~4GB. `DerivedData`·`frontend/build` 삭제 가능.
  캐시 살아있으면 릴리즈 ~1분, 콜드 빌드는 5분+.
- **Material 아이콘은 굵기 지정 불가** — 굵게 보이려면 채움(filled) 버전 사용.
