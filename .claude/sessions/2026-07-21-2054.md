### 발생한 문제와 해결책

- **문제**: 캘린더 위 일정 바는 오버레이(Positioned)라 롱프레스 제스처를 자체적으로 받기 어려움
  - **해결**: 바를 `IgnorePointer`로 감싸 터치를 캘린더로 통과시키고,
    롱프레스는 `TableCalendar.onDayLongPressed`에서 일괄 처리.
    `onDayLongPressed`는 캘린더가 렌더되는 **3곳 모두**에 동일하게 연결
- **문제**: 바텀시트에서 항목을 고른 직후 곧바로 다이얼로그를 열면 Navigator 충돌 소지
  - **해결**: `Navigator.pop` 후 `addPostFrameCallback` + `mounted` 검사로 다음 프레임에 다이얼로그 오픈
- **문제**: 셀 반복 일정이 등록 구간과 반복 회차 양쪽에서 두 번 그려질 수 있음
  - **해결**: 반복 루프에서 `start~end` 구간에 속하는 날짜는 `continue`로 건너뜀
- **문제(iOS 설치)**: `flutter install`/`flutter run`은 앱을 삭제 후 재설치해 로컬 데이터가 날아감
  - **해결**: `flutter build ios --release`로 `.app`만 만들고
    `xcrun devicectl device install app`으로 덮어쓰기 설치 → 데이터 보존 확인

- **캘린더는 화면에 3번 렌더**된다(home_screen 내 3곳). 콜백을 하나만 고치면 특정 상태에서만 동작한다 — 반드시 3곳 모두 수정
- 일정 수정 진입은 `_openScheduleEdit({litten, roomSchedule})` 하나로 통일. 새 진입점은 여기에 붙일 것
- **아이폰 설치**: `flutter build ios --release` → `xcrun devicectl device install app`.
  `flutter install`은 데이터 초기화되므로 금지
- 백엔드 컴파일은 `./build.sh clean package -DskipTests` (기본 mvn은 JDK 25로 Lombok 깨짐)
- 실제 DB는 원격 `www.litten7.com`. 로컬 `litten7`은 껍데기
- 릴리즈 빌드마다 3~4GB 소모 — 디스크 부족 시 `DerivedData`·`frontend/build` 삭제 가능
- Material 아이콘은 굵기 지정 불가 — 굵게 보이려면 채움(filled) 버전 사용
