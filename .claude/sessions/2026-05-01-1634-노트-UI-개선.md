### 발생한 문제와 해결책

| 문제 | 원인 | 해결 |
|------|------|------|
| 녹음 중 탭 복귀 시 `Exception: 녹음 중에는 리튼을 변경할 수 없습니다` | `_onAppStateChanged`에서 `selectLitten` 호출 시 녹음 중 예외 발생 | `if (AudioService().isRecording) return;` 가드 추가 |
| STT 시작해도 AllFilesTab 녹음 버튼 미반영 | STT도 `AudioService.startRecording()` 호출하지만 상태 동기화 누락 | `_onAudioStateChanged`에서 `_audioService.isRecording` 직접 참조로 완전 동기화 |
| PageView 탭 전환 후 WritingScreen 초기화 | PageView가 비인접 탭(0↔2) 전환 시 위젯 destroy | `AutomaticKeepAliveClientMixin` + `super.build(context)` 적용 |
| 'all' 탭이 사라짐 | 사용자가 실수로 bottomLeft→topLeft 드래그 후 bottomLeft 영역 empty → 비표시 | SharedPreferences 기본값을 `topLeft`로 수정, 도킹 설정으로 하단 영역 제어 |

### 중요 아키텍처 결정

- **`AudioService()` 싱글톤**: 녹음 상태를 모든 위젯에서 일관되게 참조할 수 있는 단일 진실의 원천
- **`showBottomDock` 파라미터**: `DraggableTabLayout`에 도킹 여부를 props로 주입, 레이아웃 결정은 위젯 내부에서
- **`_buildSettingsSwitchItem`**: Switch 전용 설정 항목 메서드 분리 (기존 `_buildSettingsItem`과 구분)
- **`isDraggable` 필드 (`TabItem`)**: 탭별 드래그 가능 여부 개별 제어 (이번 세션에서 추가, 현재 미사용)

### 미래 개발자를 위한 팁

1. **녹음 상태 관련 작업**: `AudioService().isRecording`로 현재 녹음 여부 확인. STT와 일반 녹음 모두 같은 `AudioService` 싱글톤 사용.
2. **AllFilesTab 파일 수 업데이트**: `AppStateProvider`의 `actualTextCount`, `actualHandwritingCount`, `actualAudioCount`를 `writing_screen.dart`에서 `AllFilesTabButton`에 전달.
3. **WritingScreen 상태 유지**: `AutomaticKeepAliveClientMixin` 덕분에 PageView 탭 전환 시 상태 유지. `super.build(context)` 호출 필수.
4. **도킹 레이아웃**: `DraggableTabLayout`의 `showBottomDock` 파라미터가 `true`일 때만 세로 모드에서 하단 영역 항상 표시.
5. **자동 일정 선택 로직**: `WritingScreen._autoSelectLittenForNoteTab()` — 녹음 중에는 절대 호출되지 않도록 가드 필수.
6. **`isDraggable` 필드**: `TabItem`에 추가되어 있음. `_buildTabHeader`에서 `false`이면 `Draggable` 래퍼 없이 렌더링되고 drag_indicator 아이콘도 숨겨짐.
