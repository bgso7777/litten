### 주요 변경 사항 및 발견

1. **TableCalendar의 Builder 우선순위**
   - selectedBuilder/todayBuilder > defaultBuilder > 기본 렌더링
   - defaultBuilder가 있으면 마커가 자동으로 표시되지 않음
   - Builder를 사용하면 전체 셀 레이아웃을 직접 제어해야 함

2. **전체 화면 vs 작은 캘린더 차이점**
   - 전체 화면: SliverPersistentHeader 내부, 동적 높이 조절
   - 작은 캘린더: Transform.scale(0.95) 적용, 고정 높이
   - 두 캘린더의 calendarStyle과 calendarBuilders가 별도로 관리됨

3. **알림 캐시 구조**
   - `_notificationDateCache`: Map<String, Set<String>> 형태
   - Key: 'yyyy-MM-dd' 형식의 날짜 문자열
   - Value: 해당 날짜에 알림이 있는 리튼 ID Set
