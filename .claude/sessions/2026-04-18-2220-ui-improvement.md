### 발생한 문제와 해결책

#### 문제 1: 단일 날짜 일정 누락
- **증상**: endDate가 null인 일정들이 캘린더에 표시되지 않음
- **원인**: 코드에서 `if (schedule.endDate == null) continue;`로 필터링
- **해결**: null 체크를 제거하고 endDate가 null이면 startDate와 같은 날로 처리
- **결과**: 모든 일정이 정상적으로 표시됨

#### 문제 2: 일정 위치 오류
- **증상**: 19일의 일정 "가나다라마바사아자"가 21일 일정 아래에 표시됨
- **원인**: 행(row) 기반 그룹화로 인해 3번째 행의 모든 일정이 함께 배치됨
- **해결**: 셀(cell) 기반 그룹화로 변경하여 각 날짜별로 독립적으로 관리
- **결과**: 각 일정이 올바른 날짜 아래에 표시됨

#### 문제 3: 일정 텍스트 가독성
- **증상**: 일정 바의 텍스트가 충분히 굵지 않음
- **원인**: FontWeight.w500 사용
- **해결**: FontWeight.bold로 변경
- **결과**: 가독성 향상

#### 문제 4: 캘린더 확장 현상 (미해결)
- **증상**: 사용자가 위로 스크롤하여 캘린더를 고정한 후 날짜를 선택하면, 캘린더가 아래로 확장되며 일정 리스트가 밀려남
- **원인**: SliverPersistentHeader의 pinned 동작 + selectDate()의 notifyListeners() 호출로 인한 위젯 리빌드
- **시도한 해결책**:
  - onDaySelected 핸들러에서 스크롤 위치 저장
  - 3단계 복원 시도: 즉시, postFrameCallback, 50ms delayed
  - 디버그 로깅 추가
- **결과**: 실패
  - 디버그 로그가 출력되지 않아 코드가 실행되지 않음을 확인
  - 핫 리로드 문제로 추정
  - 앱 재시작도 정상적으로 완료되지 않음
- **다음 단계**:

  - 스크롤 복원이 작동하지 않으면 다른 접근 방식 고려
  - SliverPersistentHeader 동작 자체를 수정하거나
  - 날짜 선택 시 리빌드를 트리거하지 않는 방법 검토

### 얻은 교훈

1. **Flutter 핫 리로드 한계**:
   - 특정 상태 관련 코드 변경은 핫 리로드로 적용되지 않을 수 있음
   - 중요한 변경 사항은 항상 완전한 앱 재시작으로 확인 필요
   - 디버그 로그가 나타나지 않으면 코드가 실행되지 않는 것

2. **SliverPersistentHeader 동작**:
   - pinned 속성이 true일 때, 위젯 리빌드 시 자동으로 확장되는 동작 있음
   - notifyListeners()가 호출되면 전체 위젯 트리가 리빌드됨
   - 스크롤 위치 복원만으로는 해결이 어려울 수 있음

3. **셀 기반 레이아웃**:
   - 그리드 형태의 UI에서 요소를 배치할 때는 행/열 인덱스를 조합한 키 사용이 효과적
   - "row_col" 형식의 키는 디버깅과 유지보수에 유리

4. **null 안전성**:
   - optional 값(endDate)을 처리할 때 기본값 제공이 필터링보다 나을 수 있음
   - 사용자 입장에서 데이터가 표시되지 않는 것보다 표시되는 것이 좋음
