### 영상 상세 팝업 본문 무한 로딩 버그 수정 (22:30 KST)
- **증상**: 노트 > 영상구독 > 채널 펼침 > 영상 제목 탭 → 팝업의 헤더(채널명/제목/일시)는 정상이지만 본문이 영원히 로딩 스피너만 표시. 서버에 요청 로그도 안 찍힘.
- **Root cause**: [api_service.dart:976](frontend/lib/services/api_service.dart#L976)의 `getYoutubeVideoDetail` URL이 `'$baseUrl/note/v1/youtube/videos/$videoId'` 로, 서버 `context-path: /litten` 누락 (다른 모든 엔드포인트는 `/litten/note/v1/...`). 결과적으로 404가 떨어져 서버 컨트롤러 미도달.
- **2차 버그**: [youtube_video_detail_dialog.dart](frontend/lib/widgets/youtube_video_detail_dialog.dart) `_pollTimer`가 `detailCache.containsKey(videoId)`만 확인 → API 실패 시 cache에 들어가지 않으므로 폴링이 영원히 지속, 다이얼로그가 무한 뺑뺑이.
- **수정**:
  1. `api_service.dart:976` URL에 `/litten` prefix 추가
  2. `youtube_video_detail_dialog.dart` 폴링 종료 조건에 `!loadingSet.contains(videoId)` 추가 → 향후 다른 API 실패 케이스에서도 무한 로딩 방지
