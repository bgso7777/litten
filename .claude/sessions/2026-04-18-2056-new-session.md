### 학습 내용 (Lessons Learned)

#### 기술적 인사이트
1. **dictation 모드의 본질적 특성**
   - STT 엔진이 이전 텍스트를 계속 재인식하며 미세하게 변경
   - 토큰 수준의 비교는 불가능, 문자열 길이 기반 접근만 가능
   - 엔진 레벨의 제약은 앱에서 우회할 수 없음

2. **DOM 조작 vs API 호출**
   - Summernote의 `insertText` API는 연속 호출 시 커서 에러 발생
   - 직접 DOM 조작(`appendChild`, `replaceChild`)이 더 안정적
   - 유지보수 중단된 라이브러리는 API보다 DOM 직접 조작이 나음

3. **실시간 처리의 중요성**
   - 누적 방식은 데이터 꼬임과 커서 문제 야기
   - isFinal 이벤트 기반 실시간 확정이 훨씬 안정적
   - 과거 실패가 현재 해결책을 막는 이유가 되어선 안 됨
