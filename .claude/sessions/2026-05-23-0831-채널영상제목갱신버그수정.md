- `YoutubeService.java:255` 에서 `existsByVideoId(videoId)` 가 true이면 제목 변경 없이 완전히 스킵

### 수정 계획
- `existsByVideoId` → `findByVideoId`로 변경하여 기존 영상 엔티티 조회
- 기존 영상이면 제목이 변경된 경우에만 DB 업데이트 후 스킵
- 신규 영상이면 기존 로직(`processNewVideo`) 실행

### 주요 성과

#### 1. YouTube 자막 추출 시스템 완전 재설계
- **이전**: YouTube timedtext API 직접 호출 → YouTube BotGuard 차단으로 실패
- **이후**: 3단계 파이프라인
  1. Frontend HTTP 직접 추출 시도
  2. Backend yt-dlp / Supadata 1:1 라운드로빈 (짝수 요청 = yt-dlp 먼저, 홀수 = Supadata 먼저)
  3. 실패 시 "내용을 불러올 수 없습니다." 메시지 표시 (WebView 제거)

#### 2. yt-dlp 서버 연동
- 서버에 yt-dlp 설치 (`/usr/local/bin/yt-dlp`)
- JSON3/VTT/TTML 포맷 자동 파싱
- YouTube 쿠키 파일 지원 (`--cookies` 옵션)
- IP 차단 대비 `YOUTUBE_YTDLP_ENABLED=false` env var로 즉시 비활성화 가능

#### 3. Supadata API 연동
- 엔드포인트: `GET https://api.supadata.ai/v1/youtube/transcript?videoId={id}&lang=ko&text=true`
- API 키: `x-api-key` 헤더
- JSON 응답 ObjectMapper 파싱 (regex StackOverflowError 방지)

#### 4. API 키 통합 관리 (`ApiKeyProperties`)
- yml/환경변수 우선, 없으면 `/home/backend/api-key` YAML 파일 자동 로드
- supadata, claude, openai 3개 키 통합 관리
- SummaryService, OpenAiSummaryService에 적용

#### 5. no_transcript 재시도 스케줄러
- 자막 없는 영상을 30분마다 배치 재시도 (20개씩)
- `YoutubeScheduler.retryNoTranscriptVideos()` @Scheduled

#### 6. 채널 영상 제목 갱신 버그 수정 (세션 원래 목표)

### 발생한 문제와 해결책

| 문제 | 원인 | 해결 |
|------|------|------|
| YouTube BotGuard 차단 | timedtext API 직접 호출 차단 | yt-dlp(PoToken 내장) + Supadata 대체 |
| yt-dlp "No such file or directory" | 서버에 미설치 | `sudo curl -L -o /usr/local/bin/yt-dlp` |
| yt-dlp "Did not get any data blocks" | VTT 파일 생성됐으나 JSON3만 탐색 | `--sub-format json3/vtt/ttml` + VTT 파서 추가 |
| `>` 등 유니코드 이스케이프 표시 | JSON 이스케이프 미변환 | `unescapeUnicode()` 메서드 추가 |
| HttpLoggingFilter ERROR 스팸 | 긴 body 잘린 JSON 파싱 실패 → log.error | log.debug로 변경 |
| Supadata StackOverflowError | 긴 자막 텍스트에서 regex 재귀 스택 초과 | `ObjectMapper.readValue()` 교체 |
| unescapeUnicode `$`/`\` 오동작 | appendReplacement 특수문자 미처리 | `Matcher.quoteReplacement()` 적용 |
| APK 설치 실패 | 앱 실행 중 | `am force-stop` 후 재설치 |
