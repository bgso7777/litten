### 교훈 / 미래 개발자 팁
- 대규모 리네임은 sed 전 `reminder` 같은 부분일치/동음이의(youtube autoRemind) 먼저 점검. 토큰 순서(복수형/대문자→일반) 중요.
- `ddl-auto: none`이라 엔티티 @Table/@Column 변경은 자동 반영 안 됨 → 반드시 마이그레이션 SQL 동반. 컬럼명뿐 아니라 **데이터 값**(prompt type='quiz', 프롬프트 본문 마커)까지 확인.
- API 계약(엔드포인트+JSON 키)은 프론트·백엔드 동일 규칙으로 동시 변경해야 함. 적용 순서: 마이그레이션 → 백엔드 WAR → 프론트.
- 리마인드↔퀴즈는 이제 같은 개념(인사이트 탭=요약+퀴즈, 메인탭 라벨은 "리마인드"). 백엔드 프롬프트의 "Q&A에서의 인사이트"·마케팅 html·회원 이메일의 insight는 무관하니 건드리지 말 것.
- 검증 1순위: 텍스트/영상 요약 1건 → 리마인드(인사이트) 미완료 퀴즈에 항목 생성되면 전체 파이프라인 정상.
