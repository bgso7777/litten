// text_tab.dart - AppStateProvider 리스너
void _onAppStateChanged() {
  if (!mounted) return;
  final appState = Provider.of<AppStateProvider>(context, listen: false);
  if (_lastSTTActiveState == true &&
      appState.isSTTActive == false &&
      _isListening) {
    _stopListening();
  }
  _lastSTTActiveState = appState.isSTTActive;
}

### 발생한 문제와 해결책

#### 문제 1: STT 상태 변화 감지 실패
- **증상**: 녹음 시작 시 STT가 중단되지 않음
- **원인**: `build()` 메서드는 다른 탭에 있을 때 실행되지 않음
- **해결**: AppStateProvider 리스너 패턴으로 변경
- **교훈**: 탭 기반 UI에서는 리스너 패턴이 더 효과적

#### 문제 2: Widget 상태 손실
- **증상**: 백그라운드 전환 시 위젯이 dispose됨
- **원인 1**: `_loadFiles()`가 편집 상태 무시하고 재로드
- **원인 2**: `WritingScreen`이 매번 새 탭 위젯 생성
- **원인 3**: `MaterialApp`이 rebuild 시 home 위젯 재생성
- **해결**: 3단계 방어 (편집 체크 + GlobalKey + 캐싱)
- **교훈**: Flutter 위젯 생명주기는 다층 방어가 필요

#### 문제 3: iOS 코드 서명
- **증상**: `flutter install` 실패 (코드 서명 없음)
- **원인**: `--no-codesign` 플래그 사용
- **해결**:
  1. `flutter build ios --release` (서명 포함)
  2. `xcrun devicectl device install app`
  3. `xcrun devicectl device process launch`
- **교훈**: iOS 실기기 배포는 항상 서명 필요

#### 문제 4: BuildContext async gap 경고
- **증상**: async 작업 후 BuildContext 사용 시 경고
- **해결**: `if (!mounted) return;` 체크 추가
- **교훈**: async 작업 전후로 mounted 체크 필수
