### 발생한 문제와 해결책

#### 문제 1: Hot Reload 미작동
- **증상**: 코드 변경 후 시뮬레이터에 반영 안 됨
- **원인**: StatefulWidget의 구조적 변경 시 Hot Reload 한계
- **해결책**:
  - `killall -9 Runner`로 앱 강제 종료
  - `pkill -9 -f "flutter run"`로 프로세스 종료
  - 완전 재실행으로 변경사항 반영
- **재발 방지**: 구조적 변경 시 항상 완전 재실행 권장

#### 문제 2: diameterRatio 0.8로 설정 시 시간 안 보임
- **증상**: 스크린샷에서 현재 선택된 시간만 보이고 위아래 시간 전혀 안 보임
- **원인**: diameterRatio가 너무 작아서 스크롤 뷰가 너무 평평함
- **해결 과정**:
  1. 1.5 → 0.8 (빠른 반응 목적, 실패)
  2. 0.8 → 1.2 (가시성 회복)
  3. 1.2 → 1.6 (최종, 30% 추가 증가)
- **해결책**: diameterRatio 1.6으로 조정하여 가시성과 반응성 균형
- **교훈**:
  - diameterRatio는 0.8 미만 비권장 (가시성 문제)
  - 1.5~2.0 범위가 적절
  - 사용자 피드백으로 점진적 조정 필요

#### 문제 3: 노트 + 버튼 애니메이션 시 아래로 쳐짐
- **증상**: 메뉴 열릴 때 + 버튼이 아래로 약간 내려가 보임
- **해결 과정**:
  1. bottom: 4 (초기)
  2. bottom: 16 (캘린더 맞춤)
  3. bottom: 24 (더 올림)
  4. bottom: 28 (최종, 안정적)
- **해결책**: bottom 값을 28로 설정하여 애니메이션 시에도 안정적
- **교훈**: 애니메이션 포함 UI는 여유 공간 필요

### 얻은 교훈

1. **Flutter Hot Reload의 한계**
   - StatefulWidget의 구조적 변경은 Hot Reload로 반영 안 됨
   - build() 메서드 내 UI 변경만 Hot Reload 가능
   - initState(), dispose() 변경 시 Hot Restart 필수

2. **ListWheelScrollView 튜닝**
   - `diameterRatio`: 1.5~2.0 권장 (가시성과 반응성 균형)
   - `itemExtent`: 전체 높이의 30~40% 권장
   - `perspective`: 0.01~0.005 (작을수록 평평)

3. **UI 간격 조정 원칙**
   - 관련 요소 간: 작은 간격 (2-4)
   - 섹션 간: 중간 간격 (8-12)
   - 큰 블록 간: 큰 간격 (16-24)

4. **사용자 피드백 기반 반복 개선**
   - 초기 구현: 빠른 프로토타입
   - 사용자 테스트: 실제 사용성 확인
   - 점진적 조정: 피드백 반영하여 미세 조정
   - diameterRatio 조정 과정이 좋은 예시

5. **코드 정리의 중요성**
   - 불필요한 요소 제거 (시계 아이콘, 하단 시간 표시)
   - 일관된 스타일 적용 (모든 입력창 labelText)
   - 명확한 주석 작성
