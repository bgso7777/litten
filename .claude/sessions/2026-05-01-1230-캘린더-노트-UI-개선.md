**주요 발견/교훈**:
- `notifyListeners()`는 `protected` — 외부 호출 시 public wrapper 필요
- `Timer.periodic` 대신 자기재귀 `Timer`를 사용하면 간격을 동적으로 변경 가능
- `firedNotifications` 기반 뱃지는 알림이 실제 발동해야만 표시 — 스케줄 기반으로 대체 필요
- `_getNextTriggerTime()` onDay 케이스가 strict `>` 비교 → 동일 분 생성 시 null 반환 버그
- 하단 탭 뱃지와 캘린더 날짜 도트는 별도 로직 (전자는 NotificationService, 후자는 HomeScreen)
- 노트탭 `OverscrollNotification`은 Android에서 동작 안 함 → ScrollController 방식으로 대체

**미래 개발자 팁**:
- 뱃지 해제 흐름: 클릭 → `badge_dismissed_litten_ids` 저장 OR `StoredNotification.isAcknowledged` → `notifyBadgeChange()` → `_loadNotificationDates()` → `updateScheduleBadgeCount(0)` → 뱃지 사라짐
- `_scheduleChipRefresh()`는 매번 `_getScheduleHint()`를 재계산해 인터벌을 결정하므로 상태 변수 불필요
- `ignoreSelectedDate` 파라미터로 노트탭/캘린더탭 필터 분리 가능
