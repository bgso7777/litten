- 일정 달린 리튼은 `pullSchedules`가 최소 리튼을 자동 생성(schedule_sync_service.dart:141)

## 동기화 버그 수정 (2026-06-13) — 녹음/필기/첨부 업로드 누락 + 중복
- **증상**: 아이폰(현재 버전 재설치, 스탠다드→실제 로그인됨=프리미엄) 로그인 시 텍스트(노트)만 동기화, 녹음/필기/첨부 누락. 새 녹음은 정상 올라가나 아이폰에 중복 2개 생김.
- **원인1(업로드 누락)**: 녹음/필기/첨부는 절대경로 저장·사용 → iOS 재설치로 컨테이너 경로 무효 → file.exists() 실패. 텍스트/필기PNG는 {localId}.ext+appDir 재구성이라 안전.
- **원인2(중복)**: 녹음 원본=`{표시명}.m4a`(id=타임스탬프≠파일명), 다운로드본=`{localId}.m4a` → 디스크 2파일 → getAudioFiles 디렉토리 스캔이 2개로 표시.
- **수정(sync_service.dart)**:
  - `_saveDownloadedFile`: 오디오 다운로드본을 cloudFileName(원본명)으로 저장
  - `_resolveLocalPath` 헬퍼: 경로 무효 시 appDir 기준 재구성. `_uploadUnsyncedFilesForLitten`(오디오/필기/첨부) + `_getFilePath`에 적용
- **검증**: 안드로이드 회귀 없음(동기화 정상) 확인. iOS는 경로 고정이라 재현 불가 → 맥/Xcode iOS 빌드 재설치 후 E2E 검증 필요. 이미 생긴 중복은 자동 정리 안 됨(수동 삭제).

### 발견 사항 (비자명)
- **온보딩 간헐 스킵 원인**: 첫 실행 시 알림 권한 다이얼로그가 온보딩 화면 위에 겹쳐 뜨고, 다이얼로그를 닫는 과정에서 입력이 관통되어 온보딩이 의도치 않게 완료(`completeOnboarding` 호출)됨.
- **회피법**: 테스트 전 `adb shell pm grant com.litten.app android.permission.POST_NOTIFICATIONS`로 권한 선부여 → 다이얼로그 없이 온보딩 첫 페이지에서 정상 정지.
- 깨끗한 재현 절차: `am force-stop` → `pm clear` → `pm grant POST_NOTIFICATIONS` → 실행.
- `is_app_initialized`(SharedPreferences) 키 유무로 첫 실행 판정 (`completeOnboarding`에서만 set).
