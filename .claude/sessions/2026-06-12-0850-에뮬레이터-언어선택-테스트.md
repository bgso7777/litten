## 동기화 분석 결론 (스탠다드 플랜 기준)
- **일정(schedule)**: 동기화됨 — `ScheduleSyncService`는 로그인만 요구(프리미엄 무관)
- **파일**: 동기화 안 됨 — `SyncService._canSync`가 `isPremium`(=premium) 요구. 스탠다드는 서버 보관 대상 아님 (정상 설계)
- 따라서 줄곧 스탠다드였던 계정은 파일이 서버에 없음 → 새 기기엔 **일정만** 내려오는 게 정상
- 로그인 트리거: 로그인 전환 시 `uploadAllSchedules()`→`pullSchedules()`; 파일 동기화는 `isPremiumPlusUser=false`라 스킵
- 일정 달린 리튼은 `pullSchedules`가 최소 리튼을 자동 생성(schedule_sync_service.dart:141)

### 발견 사항 (비자명)
- **온보딩 간헐 스킵 원인**: 첫 실행 시 알림 권한 다이얼로그가 온보딩 화면 위에 겹쳐 뜨고, 다이얼로그를 닫는 과정에서 입력이 관통되어 온보딩이 의도치 않게 완료(`completeOnboarding` 호출)됨.
- **회피법**: 테스트 전 `adb shell pm grant com.litten.app android.permission.POST_NOTIFICATIONS`로 권한 선부여 → 다이얼로그 없이 온보딩 첫 페이지에서 정상 정지.
- 깨끗한 재현 절차: `am force-stop` → `pm clear` → `pm grant POST_NOTIFICATIONS` → 실행.
- `is_app_initialized`(SharedPreferences) 키 유무로 첫 실행 판정 (`completeOnboarding`에서만 set).
