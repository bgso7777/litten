### 중요 발견 / 교훈
- **근본 원인은 iOS 업로드 fileName의 .m4a 누락** — 이게 확장자 없는 파일을 만들어 연쇄 버그 유발. AOS는 다운로드 시 .m4a 강제로 방어했으나, iOS 빌드 최신화가 진짜 해결.
- 파일 식별/저장은 **표시명·절대경로가 아니라 안정 키(localId)로 통일**해야 견고.
- 시각(updatedAt)을 타임존 없는 로컬 문자열로 저장 → 기기 타임존 다르면 정렬 꼬임. **UTC 통일이 근본**(미적용).
- 에뮬 테스트 환경 팁: `pm grant POST_NOTIFICATIONS`로 온보딩 권한 관통 회피, 태블릿 AVD는 GPU host 필수, 에뮬 타임존 KST로 맞출 것.

### 설정/환경 변경
- 태블릿 AVD config.ini: GPU host, RAM 3072M, 4코어, 32bit, heap 512M
- 두 에뮬 타임존: Asia/Seoul (setprop persist.sys.timezone)
- 종속성 변경 없음

### 미래 개발자를 위한 팁 / 미완료
1. **iOS 빌드 최신화 필수**(맥): git pull 후 flutter run으로 실기기 검증. 업로드 fileName에 .m4a 확인.
2. 이미 유실된 녹음(1781344450525 등)은 서버에도 없어 복구 불가 → iOS에서 재녹음/재업로드 필요.
3. 확장자 없는 고아 파일("녹음 26..." 등) 디스크 잔재 — 화면 무해하나 1회성 정리 로직 추가 검토.
4. 근본 개선 후보: 시각 UTC 통일 저장(타임존 무관), 파일 경로 전면 localId 기반.
5. 관련 메모리: [[sync-ios-reinstall-path-bug]]
