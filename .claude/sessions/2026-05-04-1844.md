### 구현된 기능 상세

#### 1. STT 녹음파일 배지 수정
- **원인**: Android에서 파일 경로가 symlink 형태로 제공되어 `storedByPath` 맵 조회 실패
- **해결**: `storedByName` (파일명 기반) 맵 추가, `path` 조회 실패 시 `fileName`으로 fallback
- **파일**: `audio_service.dart`, `audio_file.dart`
- **핵심 코드**: `stored.copyWith(filePath: file.path)` — 실제 경로로 정규화

#### 2. STT 자동 스크롤 수정
- **원인**: `editable.scrollTop = editable.scrollHeight` 방식이 html-editor-enhanced에서 신뢰할 수 없음
- **해결**: `span.scrollIntoView({behavior: 'instant', block: 'nearest'})` JavaScript 방식으로 변경
- **파일**: `text_tab.dart` (2곳: `appendPartialTranscript`, `convertPartialToFinal`)

#### 3. 요약 리마인드 칩
- **신규 파일**: `frontend/lib/widgets/common/summary_reminder_chip.dart`
- **Painter**: `_ConcaveChipTopPainter` — 상단 모서리 오목 (캘린더 `_ConcaveChipPainter`와 동일 형태)
- **디자인**: fillColor alpha 0.08, 아이콘 `keyboard_arrow_up` 오른쪽 배치, 크기 24px
- **배치**: `WritingScreen` → `Column([Expanded(draggableTabLayout), SummaryReminderChip()])`
- **현재 상태**: 플레이스홀더 (TODO: 실제 요약 리마인드 데이터 연결)

#### 4. 캘린더 FAB 위치 조정
- **변경**: `Transform.translate(Offset(0, -12))` → `Offset(0, -48)`
- **이유**: 노트탭 FAB는 `Positioned(bottom: 28)` + chip(36px) + nav(56px) = 120px 위에 있고, 캘린더 FAB는 기본 margin(16px) + nav(56px) = 72px에 위치해 36px 차이 발생
- **파일**: `main_tab_screen.dart`

### 얻은 교훈

1. **Android 경로 신뢰 금지**: Android에서 `file.path`는 symlink일 수 있어 저장된 경로와 불일치 가능 → 파일명 기반 secondary lookup 항상 준비 필요
2. **WebView JavaScript 스크롤**: `scrollTop` 직접 조작보다 `scrollIntoView()`가 더 신뢰할 수 있음
3. **FAB 위치 계산**: Scaffold FAB는 `kFloatingActionButtonMargin(16px)` + nav bar 높이로 위치 결정. 커스텀 Positioned FAB와 일치시키려면 차이를 직접 계산해야 함
4. **칩 디자인 일관성**: 같은 역할을 하는 칩은 Painter 클래스, 색상 alpha, 패딩, 아이콘 크기 모두 통일해야 함
