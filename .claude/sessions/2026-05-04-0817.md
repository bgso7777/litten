### 수정된 버그 상세

#### 1. 백엔드 물리 파일 미삭제 (`CloudFileService.java`)
- **증상**: 파일 삭제 시 DB에 `isDeleted=true`만 기록, 디스크 파일은 계속 남아있음
- **원인**: `deleteFile()` 메서드에 `localStorageService.delete()` 호출 누락
- **수정**: 소프트 딜리트 후 `localStorageService.delete(cloudFile.getFilePath())` 추가, 물리 삭제 실패 시 DB 삭제는 유지하고 warn 로그만 출력

#### 2. 삭제된 파일 재다운로드 (`sync_service.dart`)
- **증상**: 로컬에서 삭제한 파일이 동기화 시 클라우드에서 다시 다운로드됨
- **원인**: `_bidirectionalSync()`에서 `processOfflineQueue()`(삭제 큐 처리)가 클라우드 파일 목록 조회 **이후**에 실행 → 삭제되지 않은 상태의 클라우드 파일 목록 기준으로 다운로드
- **수정**: `processOfflineQueue()` 호출을 클라우드 파일 목록 조회 **이전**으로 이동

#### 3. 파일명 중복 충돌 (`sync_service.dart`)
- **증상**: 같은 분(minute)에 생성된 두 파일이 동일한 이름(`필기 2605040702`)을 가짐
- **원인**: `_dateBasedName()`이 `YYMMDDHHmm`(분 단위)까지만 생성
- **수정**: 초(seconds) 추가로 `YYMMDDHHmmss` 형식으로 변경, `_uniqueName()` 헬퍼 추가로 동일명 시 `(2)`, `(3)` 접미사 자동 부여

#### 4. 파일 수정 동기화 미작동 (`sync_service.dart`)
- **증상**: 파일 생성·삭제는 동기화되나 내용 수정은 클라우드에 반영 안 됨
- **원인**: `_uploadUnsyncedFilesForLitten()`이 `cloudId == null`(신규 파일)만 처리, 수정된 기존 파일 무시
- **수정**: `_isModifiedLocally()` 헬퍼 추가 (로컬 `updatedAt` > 클라우드 `updatedAt` + 2초 여유), 텍스트·필기 파일 루프에 수정 감지 후 `updateFile()` 호출 로직 추가

#### 5. 음성 메모 오디오 파일 미업로드 (`text_tab.dart`)
- **증상**: STT 음성 메모 종료 시 텍스트는 클라우드로 올라가지만 오디오 파일은 업로드 안 됨
- **원인 (경쟁 조건)**:
  1. `_stopRecordingWithSTT()`가 `await stopRecording()` 중 위젯 dispose 발생
  2. `dispose()`가 `cancelRecording()` 호출 → **녹음 파일 삭제**
  3. `_stopRecordingWithSTT()` 재개 시 `setState() called after dispose` 예외 → catch 블록으로 빠져 업로드 코드 미실행
- **수정**:
  - `_isStoppingRecording` 플래그 추가
  - `dispose()`에서 `_isStoppingRecording == true`면 `cancelRecording()` 건너뜀 (파일 보존)
  - `_stopRecordingWithSTT()`의 모든 `setState()` 호출에 `if (mounted)` 체크
  - `try/finally`로 플래그 항상 초기화 보장
  - 파일 저장(`LittenService`) · 클라우드 업로드(`SyncService`)는 `context` 불필요하므로 mounted 무관하게 실행
