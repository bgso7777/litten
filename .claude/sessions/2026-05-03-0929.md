## 주요 기술 포인트 (미래 개발자를 위한 팁)

### 오디오 파일 메타데이터 관리
`AudioService.getAudioFiles()`는 파일시스템 `.m4a` 스캔 후 `FileStorageService.loadAudioFiles()`의 저장된 메타데이터와 **filePath 기준으로 병합**한다. 새 파일은 `stopRecording()` 시 즉시 저장된다. 실제 파일이 없는 메타데이터는 자동 정리된다.

### 필기 파일 저장 구조
- **캔버스 파일** (단일 페이지, `HandwritingType.drawing`): 실제 드로잉은 `{id}_drawing.png`에 저장. `imagePath`는 `/temp/placeholder.png`로 의미 없음.
- **PDF 변환 파일** (다중 페이지, `HandwritingType.pdfConvert`): 각 페이지는 `{mainId}_page_{N}.png`, 드로잉 레이어는 `{mainId}_page_{N}_drawing.png`.
- 클라우드 업로드 시 캔버스 파일은 `{id}_drawing.png` 경로를 직접 계산해야 함.

### SyncService 콜백 패턴
```dart
// AppStateProvider 초기화 시
SyncService.instance.init(_authService, onSyncStatusChanged: notifyFileListChanged);
// SyncService가 동기화 완료 → _onSyncStatusChanged?.call() → notifyFileListChanged() → 파일 목록 자동 갱신
```

### 동기화 3가지 시점
1. **로그인 시** (`syncOnLogin`): 양방향 전체 동기화
2. **앱 시작 시** (`syncOnAppStart`): 마지막 동기화 이후 변경분만 증분 동기화
3. **포그라운드 전환 시** (`syncOnForeground`): 백그라운드 5분 이상일 때만 실행

1. `getAudioFiles()`처럼 매번 새 객체를 생성하는 패턴은 동기화 메타데이터와 함께 쓸 때 반드시 영속성 레이어(SharedPreferences 등)와 병합해야 한다.
2. 파일 쓰기 전 `directory.create(recursive: true)` 확인이 필수. 누락 시 try-catch로 무음 실패한다.

4. Provider 패턴에서 SyncService → AppStateProvider로의 콜백은 순환 의존 없이 단방향으로 유지해야 한다.
