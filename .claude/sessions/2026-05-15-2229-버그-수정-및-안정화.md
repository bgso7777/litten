## 진행 상황

### 완료된 작업
- **STT/녹음 중 selectedLitten 해제 버그 방어 패치 (1차)**
  - `app_state_provider.dart` `refreshLittens()` — 갱신 리스트에서 찾지 못해도 기존 `_selectedLitten` 참조 유지.
  - `audio_service.dart` — `currentRecordingLittenId` getter 노출.
  - `text_tab.dart` — `_resolveLittenForStop()` 헬퍼 추가.
  - `text_tab.dart` build — STT/녹음 중엔 selectedLitten=null이어도 에디터 유지.

- **STT/녹음 중 리튼 잠금(lock) 글로벌 변수 도입 (사용자 요청 반영)**
  - 사용자 보고: 알림 없이 단순 백그라운드/화면꺼짐만으로도 selectedLitten이 해제됨.
  - `app_state_provider.dart`:
    - 필드 `_operationLockedLitten` + 키 `operation_locked_litten_id` (SharedPreferences 영구 저장).
    - 게터 `effectiveSelectedLitten` (selected → lock 폴백), `operationLockedLitten`.
    - 메서드 `lockLittenForOperation(litten)`, `unlockLittenForOperation()`, `_restoreOperationLockedLitten()` (콜드 스타트 복원).
    - `initializeApp()`에 락 복원 호출 추가.
  - `text_tab.dart`:
    - `_startListening()`: STT 시작 시 락 설정.
    - `_startRecordingWithSTT()`: 락이 비어 있으면 잠금 + effectiveSelectedLitten 사용.
    - `_stopListening()`: STT/녹음 모두 끝나면 락 해제.
    - `_stopRecordingWithSTT()` finally: STT/녹음 모두 끝나면 락 해제.
    - `_resolveLittenForStop`: effectiveSelectedLitten 우선 → recordingLittenId → currentTextFile.littenId 순.
    - build EmptyState 가드: `hasOperationLock`도 고려해 에디터 유지.
  - `recording_tab.dart`:
    - 녹음 시작 직전 `lockLittenForOperation` 호출, 정지 후 unlock.
    - 정지에 사용할 리튼을 effectiveSelectedLitten → recordingLittenId 폴백.
