1. **timeout은 양쪽 모두 명시**: 클라이언트와 서버 모두 명시하지 않으면 한쪽이 먼저 끊김.
2. **변수명 충돌은 명확화**: 매개변수와 필드 이름이 같으면 closure에서 의도 불명. 별도 변수로 분리.
3. **빌드 캐시 + JDK 버전**: `mvn compile`은 성공해도 `mvn clean package`는 실패할 수 있음 (Lombok+JDK 호환).
4. **PopupMenuButton 사이징**: 기본 padding이 IconButton보다 큼 → SizedBox + iconSize 명시 필수.
5. **요약 데이터 흐름**: HTML(자동 요약 마커) ↔ 순수 텍스트 변환을 명확히 관리.
6. **요약 그룹화**: 파일 단위가 아닌 "1회 요약 단위"로 묶는 것이 사용자 의도에 더 부합.

### 미래 개발자를 위한 팁
1. **요약 데이터 모델**: `TextFile.summaryHistory`는 `List<SummaryRecord>` (id, level, language 포함). 새 SummaryRecord 추가 시 RemindItem과 연결 가능.
2. **summaryGroupId 매칭**: 현재 RemindItem.summaryGroupId와 SummaryRecord 간 직접 매핑 없음. 추후 SummaryRecord에 id 필드 추가 후 동기화 권장.
3. **첫 항목에만 summaryText 저장**: RemindParser에서 `items.isEmpty` 체크로 첫 항목에만 저장. 데이터 중복 방지.
4. **그룹 키**: `target.summaryGroupId ?? 'file:${target.fileId}'` 패턴 (기존 데이터 폴백).
5. **SelectionArea**: GestureDetector의 onTap과 충돌 없음 (long-press로 선택). 텍스트 영역은 SelectionArea로 감싸기.
6. **백엔드 빌드**: `JAVA_HOME=/opt/homebrew/Cellar/openjdk@17/.../Home && mvn clean package -DskipTests`
7. **프론트엔드 호출 흐름**: SummaryDialog → ApiService.summarizeText (5분 timeout) → POST /api/summary → SummaryService → OpenAiSummaryService.summarize → OpenAI Chat Completions API
8. **리마인드 화면 구조**: SummaryReminderChip(하단 헤더) + RemindPanel(아코디언, 3 레벨)
